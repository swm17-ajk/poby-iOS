# 화면 명세

## 공통 규칙
- 가이드라인 색상: 기본 흰색, 포즈 매칭 성공 시 민트색

---

## Screen 1: 카메라 화면

### 레이아웃
- 카메라 영역: 화면 대부분 차지
- 조작 영역: 하단 고정 (일반적인 네비게이션 바가 차지하는 영역만큼)

### 컴포넌트 목록

#### 촬영 버튼
- 위치: 카메라 영역 하단 중앙
- 동작: 사진 촬영 후 디바이스 갤러리에 저장

#### 갤러리 버튼
- 위치: 조작 영역 좌측 하단
- 동작: 디바이스 갤러리 열기

#### 전면/후면 전환 버튼
- 위치: 조작 영역 우측 하단
- 동작: 전면/후면 카메라 전환

#### 화면 비율 조작 버튼
- 위치: 조작 영역 중앙
- 동작: 카메라 화면 비율 변경

#### 가이드라인 오버레이
- 조건: 가이드라인 목록에서 요소 선택 시 활성화
- 동작: 선택한 가이드라인을 카메라 영역 위에 오버레이로 표시

#### 가이드라인 목록
- 위치: 촬영 버튼 상단
- 형태: 가로 스크롤 리스트
- 초기 상태 (가이드라인 없음): '+' 버튼만 표시
- 가이드라인 있음: 가이드라인 썸네일 목록 + 오른쪽 끝에 '+' 버튼

##### 가이드라인 요소 동작
- 탭: 해당 가이드라인을 카메라 영역에 오버레이로 적용
- 길게 누르기: '삭제하시겠습니까?' 확인 모달 표시

##### '+' 버튼 동작
- 탭: 모달 팝업 표시
- 모달 내 '가이드 사진 찍기' 버튼 탭 → Screen 3으로 이동
- 모달 내 '가이드 사진 등록하기' 버튼 탭 → Screen 4로 이동

## Screen 2: 가이드라인 추출 화면

### 진입 경로
- Screen 3 (촬영 완료 후 '완료' 버튼 탭)
- Screen 4 (사진 선택 후 '완료' 버튼 탭)

### 레이아웃
- 상단바
- 사진 영역: 선택/촬영한 사진 표시

### 컴포넌트 목록

#### 로딩 상태 (추출 진행 중)
- 화면 중단에 프로세스 바 + 로딩 애니메이션 표시

#### 추출 성공 상태
- 로딩 UI 사라짐
- 사진 위에 흰색 선으로 가이드라인 표시
- MVP: 가이드라인 수정 불가
- 완료 버튼 활성화

#### 추출 실패 상태
- 에러 메시지 표시
- 완료 버튼 비활성화 유지

#### 완료 버튼
- 위치: 상단바 우측
- 활성화 조건: 가이드라인 추출 성공 후
- 동작: 추출된 가이드라인을 저장 후 Screen 1으로 이동
- Screen 1 복귀 상태: 새 가이드라인이 목록에 추가되고 적용된 상태

#### 취소 버튼
- 위치: 상단바 좌측
- 동작: 추출 데이터 삭제 후 Screen 1으로 이동
- Screen 1 복귀 상태: 아무 가이드라인도 적용되지 않은 상태

## Screen 3: 가이드라인용 사진 촬영 화면

### 진입 경로
- Screen 1 '+' 버튼 모달에서 '가이드 사진 찍기' 탭

### 레이아웃
- 카메라 영역: 화면 대부분 차지
- 조작 영역: 하단 고정 (네비게이션 바 형태)

### 컴포넌트 목록

#### 촬영 버튼
- 위치: 카메라 영역 하단 중앙
- 동작: 사진 촬영 후 재촬영 카드 모달 표시

#### 취소 버튼
- 위치: 조작 영역 좌측 하단
- 동작: Screen 1으로 이동

#### 재촬영 카드 (촬영 버튼 탭 후 표시되는 모달)
- 재촬영 버튼: Screen 3 초기 상태로 돌아감
- 완료 버튼: 촬영한 사진을 가지고 Screen 2로 이동

---

## Screen 4: 갤러리 화면

### 진입 경로
- Screen 1 '+' 버튼 모달에서 '가이드 사진 등록하기' 탭

### 레이아웃
- 상단바
- 갤러리 영역: 사진 그리드

### 컴포넌트 목록

#### 사진 그리드
- 형태: 그리드 레이아웃
- 선택 방식: 단일 선택

#### 취소 버튼
- 위치: 상단바 좌측
- 동작: Screen 1으로 이동

#### 완료 버튼
- 위치: 상단바 우측
- 활성화 조건: 사진 1장 선택 시
- 비활성화 조건: 사진 미선택 시
- 동작: 선택한 사진을 가지고 Screen 2로 이동
